fix: stabilize backend server and resolve critical startup issues

## Summary
Fixed multiple critical issues preventing the backend from starting correctly.
The server now runs stably on port 5005 with all endpoints functioning properly.

## Critical Fixes

### 1. Port Configuration (EADDRINUSE Resolution)
- Changed backend port from 5000 to 5005 in `.env`
- Updated frontend Vite proxy configuration to target port 5005
- Resolved port conflict that was preventing server startup

### 2. Logger Crash Fix
- Disabled problematic `winston-daily-rotate-file` transports in `utils/logger.js`
- Commented out file-based exception and rejection handlers
- Maintained console logging for development
- Prevents silent crashes during server initialization

### 3. Express 5 Routing Compatibility
- Replaced incompatible wildcard routes (`app.get('*')`) with middleware-based catch-all
- Fixed SPA fallback routing to work with Express 5
- Updated 404 handling for API routes

### 4. Redis Rate Limiter Fix
- Added proper Redis connection status checks in `middleware/security.js`
- Implemented graceful fallback to memory-based rate limiting when Redis unavailable
- Fixed crash when `apiRateLimiter` attempted to use disconnected Redis client
- Added defensive null checks for error handling

### 5. Health Endpoint Optimization
- Moved `/health` and `/api/ping` endpoints before middleware chain
- Ensures health checks work even if middleware fails
- Added defensive error handling in health check endpoint
